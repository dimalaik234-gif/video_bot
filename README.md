# 🎬 Video Download Bot

Телеграм-бот для скачивания видео и аудио по ссылке. Написан на **aiogram 3.x** +
**yt-dlp**. Поддерживает YouTube, TikTok, Instagram, VK, X (Twitter), Reddit,
Pinterest, SoundCloud и 1000+ других сайтов.

## ✨ Возможности

- 📹 **Видео** в качестве до 4K (выбор: Лучшее / 1080p / 720p / 480p / 360p)
- 🎵 **Аудио MP3** (извлечение звуковой дорожки)
- 🖼 **Обложка** видео
- 📝 **Субтитры** (ru/en, авто-сабы тоже)
- 📋 **Плейлисты** (до 10 видео за раз, лимит настраивается)
- ℹ️ **Подробная информация**: автор, длительность, просмотры, лайки, дата
- 📊 Прогресс-бар загрузки в реальном времени
- ⚙️ Настройки: язык (🇷🇺/🇬🇧), качество и формат по умолчанию
- 🕓 История загрузок и личная статистика
- 🛠 Админ-панель: статистика, рассылка, бан/разбан
- 🚦 Антифлуд (троттлинг) и опциональная обязательная подписка на канал
- 🗄 База данных SQLite (пользователи, история, статистика)

## 🔑 Шаг 1. Получить токен

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → задай имя и username.
2. Скопируй токен вида `123456789:AaBb...`.
3. Свой ID узнай у [@userinfobot](https://t.me/userinfobot) — он пойдёт в `ADMIN_IDS`.

## 🚀 Шаг 2. Деплой на bothost.ru

bothost.ru поддерживает Python и **сборку по Dockerfile** — это важно, потому что
для склейки видео+аудио и конвертации в MP3 нужен **ffmpeg**, а он ставится именно
через Dockerfile (он уже в проекте).

**Вариант А — через GitHub (рекомендуется):**
1. Залей этот проект в репозиторий на GitHub/GitLab.
2. На bothost.ru создай новый проект и подключи репозиторий.
3. В настройках выбери сборку по **Dockerfile** (чтобы был ffmpeg).
4. Добавь переменные окружения (см. ниже).
5. Запусти. При каждом `git push` бот пересоберётся автоматически.

**Вариант Б — загрузка кода напрямую:**
1. Создай проект, загрузи файлы.
2. Тип окружения — Dockerfile (или Python, если ffmpeg ставить не через Docker).
3. Команда запуска: `python main.py`. Зависимости — `requirements.txt`.
4. Добавь переменные окружения и запусти.

### Переменные окружения

| Переменная        | Обязательна | Описание                                  |
|-------------------|-------------|-------------------------------------------|
| `BOT_TOKEN`       | да          | Токен от @BotFather                        |
| `ADMIN_IDS`       | желательно  | ID админов через запятую                   |
| `REQUIRED_CHANNEL`| нет         | `@канал` для обязательной подписки         |
| `MAX_FILE_SIZE`   | нет         | Лимит файла в байтах (по умолч. 50 МБ)     |
| `THROTTLE_RATE`   | нет         | Пауза между действиями (сек, по умолч. 1.5)|
| `PLAYLIST_LIMIT`  | нет         | Видео из плейлиста за раз (по умолч. 10)    |

## 💻 Локальный запуск

```bash
pip install -r requirements.txt
# поставь ffmpeg: Linux apt install ffmpeg / macOS brew install ffmpeg / Windows — с сайта ffmpeg
cp .env.example .env   # впиши свой BOT_TOKEN
python main.py
```

## ⚠️ Важные нюансы

- **Лимит Telegram 50 МБ.** Бот сам проверяет размер и предупреждает, если файл
  больше. Для файлов до 2 ГБ нужен self-hosted Telegram Bot API сервер
  (тогда поднимаешь `MAX_FILE_SIZE`).
- **Ресурсы.** На бесплатном тарифе (128–256 МБ RAM) тяжёлое видео в 4K может не
  влезть в память при склейке. Для серьёзной нагрузки бери платный тариф или
  ограничь качество.
- **ffmpeg.** Без него доступны только готовые потоки video+audio (часто максимум
  720p), а MP3 отдаётся в исходном формате аудио. Используй Dockerfile из проекта.
- **Только polling**, без webhook — подходит для bothost.ru и любого шаред-хостинга.

## ⚖️ Использование

Качай только тот контент, на который у тебя есть права (свои видео, материалы из
открытого доступа, разрешённые к скачиванию и т.п.), и соблюдай условия сервисов,
с которых скачиваешь.
